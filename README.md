电商用户行为分析与购买预测

项目流程：

1.数据加载与清洗
2.探索性数据分析
3.RFM客户分群
4.商品销售分析
5.购买预测建模
6.业务建议

技术栈：

Python, Pandas, Scikit-learn,RandomForest，XGBoost, Matplotlib.......

项目概述：
这个项目分析了英国一家在线零售商店的真实销售数据，时间跨度为2010年12月至2011年12月
通过数据清洗和分析，揭示了销售规律、客户行为等重要商业洞察

数据：https://archive.ics.uci.edu/ml/machine-learning-databases/00352/Online%20Retail.xlsx

一、核心指标解析
1. 数据基本情况
- 原始数据：541,909条交易记录
- 清洗后数据：397,884条有效记录
- 数据字段：包括商品信息、销售数量、价格、客户信息等
- 通俗理解：分析了近40万条有效的购物记录
  
2. 数据清洗过程
项目对原始数据进行了重要处理：
- 删除无效客户：删除了13.5万条没有客户ID的记录
- 过滤异常数据：删除了数量或价格为负数的异常记录
- 创建新指标：计算了销售额、分析时间维度等

二、关键发现与分析
销售金额分析
- 平均每笔交易：£22.40
- 最大单笔交易：£168,469（可能是大工厂或公司采购）
- 75%的交易在£19.80以下

商业意义：大部分交易金额不大，但存在少量大额订单，说明既有日常零售也存在一些大批发业务

1.核心业务洞察，价值极度集中
- 顶级客户：仅0.4%的客户（冠军+忠实客户）贡献了12%的收入
- 核心力量：57.2%的潜力客户贡献78%收入，是该业务的基本盘
- 长尾效应：42.4%的客户（一般+流失）仅贡献10%收入
  
2.客户健康度警示

流失风险：25%客户已流失，人均价值£652.45，还是有重要挽回的价值

活跃度分层：从冠军客户的178次购买到一般客户的1.5次，差距显著

对于顶级客户（0.4%）
- 专属服务：配备专属的客户经理，提供其优先购买权
- 深度维护：定期个性化关怀，维护其高忠诚度
对于核心客户（57.2%）
- 价值提升：通过交叉销售提高客单价
- 频次优化：设计复购激励，提升其购买频率
对于风险客户（25%）
- 唤醒计划：针对性优惠（如打折或买几送一的活动），降低重新购买门槛
- 流失分析：找出流失原因，为其改进产品服务
战略价值：
- 资源优化：将有限资源聚焦于高价值客户群体
- 收入增长：通过精准营销提升核心客户价值
- 风险控制：有效降低25%的客户流失率

五、商业建议与实施策略
1.营销策略优化
- 精准时段投放：在周四和中午12-15点加大营销广告投入
- 季节性准备：提前为11月圣诞购物期大量备货和进行促销
- 客户分层营销：基于RFM分群分别有目的的制定差异化沟通策略
  
2.运营效率提升
- 智能排班：在销售高峰时段安排更多客服人员
- 库存优化：基于销售预测模型制定采购计划
- 资源分配：重点服务高价值客户群体
  
3.商品策略优化
- 畅销品聚焦：确保蛋糕架、灯具等热销商品库存充足
- 品类扩张：在包袋、灯具等优势品类增加货量
- 采购优化：基于商品销售数据制定精准采购计划
