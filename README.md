电商用户行为分析与购买预测

项目流程
1.数据加载与清洗
2.探索性数据分析
3.RFM客户分群
4.购买预测建模
5.业务建议

技术栈
Python, Pandas, Scikit-learn,RandomForest，XGBoost, Matplotlib.......

项目概述
这个项目分析了英国一家在线零售商店的真实销售数据，时间跨度为2010年12月至2011年12月
通过数据清洗和分析，揭示了销售规律、客户行为等重要商业洞察

一、核心指标解析
1. 数据基本情况
- 原始数据：541,909条交易记录
- 清洗后数据：397,884条有效记录
- 数据字段：包括商品信息、销售数量、价格、客户信息等
- 通俗理解：分析了近40万条有效的购物记录

2. 数据清洗过程
项目对原始数据进行了重要处理：
- 删除无效客户：删除了13.5万条没有客户ID的记录
- 过滤异常数据：删除了数量或价格为负数的异常记录
- 创建新指标：计算了销售额、分析时间维度等

二、关键发现与分析
1. 销售金额分析
基本统计信息：
- 平均每笔交易：£22.40
- 最大单笔交易：£168,469（可能是大工厂或公司采购）
- 75%的交易在£19.80以下

商业意义：大部分交易金额不大，但存在少量大额订单，说明既有日常零售也存在一些大批发业务

2. 地域销售分布
英国占据绝对主导地位，其次是荷兰、爱尔兰、德国等欧洲国家。

业务启示
- 英国是核心市场，应重点维护
- 其他欧洲国家有增长潜力
- 可以考虑针对性营销策略

3. 时间趋势分析
月度销售趋势
- 明显季节性：2011年11月出现销售高峰（圣诞购物季）
- 增长态势：整体销售额呈上升趋势
- 年初低谷：2011年1月销售较低（节日后效应）
- 商业价值：为库存管理、促销安排提供时间参考

每周销售模式
- 销售额排名：周四 > 周二 > 周三 > 周五 > 周一 > 周六 > 周日
- 洞察：工作日销售明显好于周末，周四是最佳销售日（可能跟我国情况不太一样哈哈）

每日销售时段
- 高峰时段：中午12点至下午3点
- 低谷时段：早上7点前和晚上8点后
- 平稳期：上午10点到下午6点
- 应用价值：指导客服进行排班、搞点促销活动

4. 客户价值分析
客户概况：
- 总客户数：4,372名
- 平均客户价值：£1,335.99
- 平均订单数：5.4次
  
重要发现：
- 客户价值较高：平均每个客户贡献超过1300英镑
- 复购率不错：平均每个客户下单5次以上
- 客户忠诚度：存在稳定的客户群体

三、商业建议
基于分析结果，我为该电商提出以下建议：

1. 营销策略优化
- 重点时段投放：在周四和中午时段加大广告投入
- 季节性准备：提前为11月销售高峰备货
- 欧洲市场拓展：在英国之外寻找增长机会

2. 运营效率提升
- 客服排班：在销售高峰时段安排更多客服人员
- 库存管理：根据销售趋势预测库存需求

3. 客户关系管理
- 高价值客户维护：重点服务贡献大的客户
- 提升复购率：通过会员计划增加客户粘性

四、项目价值总结
这个数据分析项目能够：
- 看清过去：了解销售规律和客户行为
- 把握现在：优化当前的运营策略
- 预测未来：为未来的商业决策提供数据支持

通过这样的分析，就能用用数据驱动决策，而不是凭感觉做生意，这还是挺重要的
